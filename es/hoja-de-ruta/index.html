Hoja de ruta - Desarrollo de BTG

En Bitcoin Gold, nos dedicamos a construir un mundo abierto y
colaborativo. Creemos que una criptomoneda poderosa no solo debe ser rápida y completa, sino que también debe ser
fundamentalmente segura y descentralizada. Estas creencias dan
forma a la hoja de ruta única de Bitcoin Gold. Esta página muestra el progreso del desarrollo y los planes futuros para el proyecto de Bitcoin Gold. Los artículos se actualizarán
regularmente a medida que proporcionemos informes de progreso quincenales a la comunidad.

Complete

Open Source
Library: bitcore

  • 100% 100%
Bitcore: infraestructura para construir Bitcoin Gold y aplicaciones basadas en blockchain. Compatibilidad con BTG agregada a bitcore- node, bitcore-lib y bitcore-deb. Un requisito previo para la
comprensión.

 

Open Source
Library: insight

  • 100% 100%
Servicio web Blockchain explorer con la API Insight en un nodo Bitcore. Nuestro explorador proporciona una API pública para la
interacción con la cadena principal, gratuita para individuos y
servicios profesionales (como Exodus).

Open Source Library: bitcoinjs-lib

  • 100% 100%
Funciones relacionadas con Bitcoin implementadas en JavaScript puro: permite a los desarrolladores web y de aplicaciones de todo
tipo (incluidos los intercambios) usar BTG fácilmente.

Open Source
Library: BitcoinJ

  • 100% 100%
Implementación de Java del protocolo Bitcoin de bitcoinj: bloque de
construcción crucial para los desarrolladores de proyectos Java,
como la cartera MyCelium.

Electrum Wallet
& Bech32

  • 100% 100%
ElectrumG es nuestro puerto del monedero Electrum, con amplias
funciones, soporte completo Bech32 (SegWit), funciones HW, etc. Este monedero liviano SPV se basa en nuestros servidores gratuitos ElectrumX.

Lightning Network Technical Preview

  • 100% 100%
La vista previa técnica completa de LN en testnet incluye el puerto de lnd (LIghtning Network Daemon de Lightning Labs) para BTG,
recksplorer (LN explorer) y lseed (LN bootstrap DNS).

Smart Contract
Research

  • 100% 100%
Conclusión de la investigación: el mejor enfoque es implementar un
lenguaje de contrato completo de Turing en cadenas laterales de plasma sobre la cadena principal de BTG. Plasma para BTG se
convertirá en un proyecto top 2019.

Active

LN: Mainnet

  • 0% 0%
Implementar una versión estable de BTG de lnd a mainnet. Otras
implementaciones de LN pueden seguir, incluyendo c-lightning
(Blockstream) y eclair (ACINQ)

Puerto btcd / btcsuite

  • 80% 80%
btcd: un nodo completo alternativo integrado en Go, base para el
conjunto completo de herramientas btcsuite. Muy utilizado por lnd y Neutrino (de Lightning Labs), es un requisito previo para el desarrollo de billetera LN móvil.

LN: Monedero de escritorio

  • 0% 0%
BTG LN es compatible con la excelente plataforma del monedero Zap , que proporciona una interfaz de escritorio fácil de usar. (El
soporte de LN también está llegando a Electrum).

LN: Cartera Móvil

  • 0% 0%
Proporcionar la experiencia de LN en dispositivos móviles será clave para una amplia adopción, por lo que estamos integrando BTG en RawTX (que requiere Neutrino).

Integración BTCPay de BTG Lightning Network

  • 30% 30%
BTCPay, la mejor pasarela de pago descentralizada, ya es
compatible con BTG, por lo que la extensión de BTCPay para ser
compatible con BTG LN es la siguiente.

Firmas Schnorr

  • 0% 0%
Muchas transacciones implican el envío de múltiples entradas a una única dirección de pago, y cada entrada requiere una firma. Las
firmas Schorr permiten firmar múltiples entradas con una sola firma, ahorrando espacio sustancial en la cadena de bloques.

Especificación de plasma BTG

  • 0% 0%
Este ambicioso proyecto comenzará con una especificación de protocolo, basada o en la extensión de las implementaciones de Plasma existentes.

Plasma: Implementación de la cadena principal (Mainchain)

  • 0% 0%
Las modificaciones de mainnet, incluidos los nuevos códigos de operación, deben probarse e implementarse para ser compatibles
con Plasma.

Plasma: implementación de cadena lateral

  • 0% 0%
Implementación real de cadenas laterales, respaldada por la cadena principal. Esto permite un gran ancho de banda y una rápida ejecución de código en la cadena lateral de Plasma mientras disfruta de la seguridad de los fondos en la cadena principal.

Monedero MyCelium

  • 0% 0%
La larga cartera MyCelium para Android y iOS agrega una serie de
características interesantes, como Local Trader para el intercambio de monedas. La adición de la funcionalidad BTG ampliará estas
características a la comunidad BTG (y se basa en la biblioteca BitcoinJ).

CCBN – Cross Chain Block Notarization

  • 0% 0%

A decentralized approach to thwart secret mining attacks by notarizing blocks to independent blockchains, providing a timing reference used to resolve chain splits in favor of honest mining.

Future

Soporte de investigación para activos emitidos por usuarios en la
cadena principal (mainchain)

Los activos emitidos por el usuario brindan soporte para modelos de negocio rentables para diversos tipos de servicios basados en
tokens.

Soporte a la technologia de intercambio de monedas criptográficas entre cadenas (Crosschain Atomic Swap)

La tecnología para el intercambio de monedas criptográficas entre
cadenas de bloques será compatible con BTG, pero el uso dependerá de que otros ofrezcan servicios de apuesta / oferta.

Transacciones privadas

Las transacciones privadas protegen los secretos comerciales de las empresas, así como la privacidad personal de las personas. Cuándo
y cómo apoyar una opción de privacidad sigue siendo una pregunta abierta.

Pools de mineria descentralizadas

Los pools de mineria actuales son autoridades centrales que
requieren confianza; Las piscinas descentralizadas ofrecen una alternativa más abierta y descentralizada.

Democracia de la cadena de bloques

La salud a largo plazo del proyecto BTG dependerá tanto del buen gobierno como de la capacidad de la comunidad para expresar
intenciones sin dificultades difíciles.
